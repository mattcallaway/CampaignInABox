# Campaign In A Box

**California Election Modeling Platform**

A modular, deterministic election modeling pipeline with an interactive Campaign Intelligence Dashboard.

---

## Quick Start

### Run the Pipeline
```powershell
cd "Campaign In A Box"
# With auto-detected election results file (detail.xlsx must be in data/)
python scripts/run_pipeline.py --state CA --county Sonoma --contest-slug prop_50_special --year 2025

# To override the election results path explicitly:
python scripts/run_pipeline.py --state CA --county Sonoma --contest-slug prop_50_special --year 2025 \
  --detail-path "data/CA/counties/Sonoma/votes/2025/prop_50_special/detail.xlsx"
```

### Launch the Campaign Intelligence Dashboard
```powershell
cd "Campaign In A Box"
streamlit run ui/dashboard/app.py
```

### Launch the Original App UI
```powershell
cd "Campaign In A Box"
streamlit run app/app.py
```

## Campaign Intelligence Dashboard

Introduced in **Prompt 9**, the Campaign Intelligence Dashboard provides an interactive interface for exploring pipeline outputs without rerunning the model.

### Features

| Page | Description |
|---|---|
| 🏠 **Overview** | Campaign KPIs from `STRATEGY_META.json` — precincts, doors, volunteers, win path |
| 🗺️ **Precinct Map** | Choropleth map with 5 selectable layers (requires `geopandas`). Falls back to bar chart. |
| 🎯 **Targeting** | Filtered targeting list with score/tier/universe filters + CSV export |
| 📋 **Strategy** | Strategy summary markdown + field plan + top targets + walk turfs |
| 🔬 **Simulations** | Win probability, outcome histogram, turnout scenarios, scatter chart |
| 🩺 **Diagnostics** | System health, join guard, integrity repairs, artifact status |
| 🗄️ **Data Explorer** | Browse any dataset with sort, filter, search, and CSV export |

### Performance
- Data is cached with `@st.cache_data(ttl=120)` — loads in < 3 seconds for 5,000+ precincts
- No model recomputation — reads `derived/` outputs only
- Use sidebar **Refresh Data** button to reload after a new pipeline run

### Dashboard Logging
Dashboard startup events are written to:
```
logs/ui/dashboard.log
```
Includes: timestamp, loaded run_id, datasets loaded, elapsed time.

## Pipeline Steps

The pipeline (`scripts/run_pipeline.py`) runs 30+ steps including:

1. **INGEST_STAGING** — optional data ingestion
2. **VALIDATE_GEOGRAPHY** — checks precinct shapefiles
3. **LOAD_GEOMETRY** — loads GeoJSON (requires `geopandas`)
4. **LOAD_CROSSWALKS** — MPREC↔SRPREC mapping
5. **NORMALIZE_SCHEMA** — canonicalizes column names
6. **BUILD_MODEL** — builds precinct model DataFrame
7. **INTEGRITY_ENFORCEMENT** — validates data constraints
8. **FEATURE_ENGINEERING** — computes model features
9. **UNIVERSE_BUILDING** — builds precinct universes
10. **SCORING_V2** — computes target scores
11. **SIMULATION** — Monte Carlo simulation (40+ scenarios)
12. **FORECAST_GENERATION** — scenario forecasts
13. **TURF_GENERATION** — walk turf clustering
14. **STRATEGY_GENERATOR** — produces STRATEGY_PACK
15. **GEO_VALIDATION** — validates geometry (Prompt 8.7)
16. **JOIN_GUARD_VALIDATION** — validates all table joins (Prompt 8.7)
17. **INTEGRITY_REPAIRS_WRITE** — writes repair audit artifacts (Prompt 8.7)
18. **ARTIFACT_VALIDATION** — checks 13 required artifacts (Prompt 8.7)
19. **POST_RUN_AUDIT** — HEALTHY/DEGRADED/FAIL system health (Prompt 8.7)

## Data Requirements

Place election results in the canonical path:
```
data/CA/counties/<County>/votes/<year>/<contest-slug>/detail.xlsx
```

For example:
```
data/CA/counties/Sonoma/votes/2025/prop_50_special/detail.xlsx
```

The pipeline auto-discovers this file — no `--detail-path` flag needed when using the canonical location.

Geography files (optional, enables map view):
```
data/CA/counties/<County>/geography/precinct_shapes/MPREC_GeoJSON/*.geojson
```
